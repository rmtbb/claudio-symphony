# Creating presets for Claudio Symphony

A preset is a **room** — a self-contained tonal world that responds to Claude Code hook events with a coherent set of sounds. Four ship in the box (cathedral, meadow, rainfall, koto). This doc walks you through making your own.

The bar to clear is not novelty but **cohesion**: every voice in a preset should belong to the same room. A bright kalimba next to a dusty granular pad is fine; a buzzy FM bell next to a warm piano is not.

---

## What you're building

A preset is a directory at `presets/<name>/` with three things:

```
presets/your-preset/
├── preset.json     # voice configs, event mappings, scale, interval weights
├── render.py       # numpy/pure-Python synth that writes the .wav samples
└── samples/        # per-voice subdirs of .wav files (gitignored, regenerated)
    ├── voice_a/
    │   ├── 00_m57.wav
    │   ├── 01_m61.wav
    │   └── ...
    └── voice_b/
        └── 00.wav
```

`samples/` is generated by running `render.py` once at install (and regenerated by `claudio regen <name>`). Don't check it into git.

## File 1 — `preset.json`

Full schema, with every field documented. Required fields are bold.

```jsonc
{
  // **REQUIRED.** Preset identity.
  "name": "your-preset",
  "description": "One sentence aesthetic statement. What kind of room is this?",

  // Optional. Path to a continuously-looping wav (relative to samples/).
  // null or omitted = no drone. Set this only if your preset has a sustained
  // bed (cathedral has one; meadow/rainfall/koto don't).
  "drone": null,

  // Optional default gains. Users can override via `claudio volume <0..1>`.
  // Defaults: 0.5 / 0.45. Bryan's runtime tunings tend to sit at 0.55 / 0.0.
  "master_gain": 0.55,
  "drone_gain": 0.0,

  // Optional. Pitch classes the Markov picker pulls toward every ~7 notes.
  // Default [9, 4] = A and E (A is the project root since A=432). Override
  // for non-A-rooted presets, e.g. [4, 11] for E Phrygian.
  "phrase_roots": [9, 4],

  // Optional. Per-preset override of the global interval weight table in
  // event.py (_INTERVAL_WEIGHTS). Keys are semitones (as JSON strings),
  // values are weights. Use this for non-major-family scales — a koto-style
  // In Sen scale, for example, wants the half-step strongly weighted.
  // Omit to use the global default tuned for pentatonic/Lydian/major.
  "interval_weights": {
    "0": 0.00, "1": 0.20, "2": 1.50, "3": 1.40, "4": 1.20,
    "5": 1.00, "6": 0.00, "7": 1.00, "8": 0.40, "9": 1.10,
    "10": 0.30, "11": 0.20, "12": 0.80
  },

  // Optional. List of MIDI pitches in the preset's scale. Required only if
  // any voice opts into tonal-overlay mode (see below). Lets the picker
  // walk noise-burst voices (wood, bird, percussive taps) onto scale pitches
  // via afplay rate-shift.
  "scale_pitches": [57, 59, 61, 64, 66, 69, 71, 73, 76, 78, 81],

  // Optional. Per-preset quantization override. Same shape as the global
  // `quant` block in config.json. Lets a slow preset like rainfall ship
  // with a 60 BPM half-note grid even if the user has the global at 120.
  "quant": { "enabled": false, "bpm": 60.0, "grid": 2.0 },

  // **REQUIRED.** Voice definitions. Keys are voice names, values are configs.
  "voices": {
    "pluck": {
      "dir": "pluck",        // **REQUIRED.** Subdir under samples/.
      "mioi": 0.5,           // **REQUIRED.** Min interval between triggers (seconds).
      "gain": 0.55,          // **REQUIRED.** Per-voice level (0..1).
      "rate_jitter": true,   // Optional. ±7c micro-detune per trigger.
                             //   Use only on plucked / percussive voices.
                             //   NEVER on bell/bowl/pad/bloom/sparkle/drone.
      "tonal_anchor_midi": 69, // Optional. Tonal-overlay mode for noise-burst voices.
                               //   The MIDI the rendered sample is tuned to.
                               //   Requires preset.scale_pitches.
      "tonal_range": 12        // Optional. Max ± semitones of shift. Default 12.
    },
    "bell": { "dir": "bell", "mioi": 4.0, "gain": 0.50 }
    // ...
  },

  // **REQUIRED.** Map hook events to voices. See "Event mapping" below.
  "events": {
    "PreToolUse":       { "default": "pluck", "by_tool": { "Bash": "breath" } },
    "PostToolUse":      { "default": "bell", "on_failure": "low-bell" },
    "Stop":             { "default": "pad" },
    "SessionStart":     { "default": "sparkle" },
    "SessionEnd":       { "default": "pad" },
    "SubagentStop":     { "default": "bell" },
    "UserPromptSubmit": { "default": "shimmer" },
    "Notification":     { "default": "bell" },
    "PreCompact":       { "default": "pad" }
  }
}
```

### Voice anatomy

Pick voice names that describe **what they sound like**, not what fires them. `pluck`, `mallet`, `bell`, `bowl`, `breath`, `bird`, `wood`, `kalimba`, `chime`, `koto`, `mokugyo`, `shimmer`, `sparkle`, `bloom`, `pad`, `swell`, `drop`, `tap` are all good. `error_sound`, `tool_voice` are bad — they couple to triggers, which the user can remap.

### Event mapping

Every Claude Code hook event can map to a voice (or `null` for silence). The router resolves in this order: `events.<event>.by_tool.<tool>` → `events.<event>.on_failure` (PostToolUse only) → `events.<event>.default`. Omit any field to fall through.

Common mapping shape:

| Event              | Suggested voice character                                       |
|--------------------|-----------------------------------------------------------------|
| `SessionStart`     | Bright opening — sparkle / cluster / arpeggio. Heard once.      |
| `UserPromptSubmit` | Soft tick — wood / breath / shimmer. Frequent.                  |
| `PreToolUse`       | Brief texture — air, breath, tap. **Most frequent event.**      |
| `PostToolUse`      | Resolution — pluck / mallet / kalimba. Frequent.                |
| `PostToolUse` failure | Different timbre — bell, chime. Distinct from success.       |
| `SubagentStop`     | Tuned bell or chime.                                            |
| `Stop`             | Long bloom / swell / pad. **The "you finished" sound.**         |
| `SessionEnd`       | Same family as Stop, slightly more final.                       |
| `Notification`     | Mid-attention — chime / bell.                                   |
| `PreCompact`       | Sustained — bloom / pad. Rare event.                            |

## File 2 — `render.py`

The renderer is pure Python + numpy. It imports synthesis primitives from the project-root `synth.py`, generates per-voice WAV samples, and writes them under `samples/<voice_dir>/`.

### Skeleton

```python
#!/usr/bin/env python3
"""
<your-preset> — sample renderer.

Aesthetic: <one paragraph stating the room you're building>
Tonal world: <scale + tuning rationale>
Voices: <one line per voice describing its sonic role>
"""
import sys, math
from pathlib import Path
import numpy as np

HERE = Path(__file__).resolve().parent
ROOT = HERE.parent.parent
sys.path.insert(0, str(ROOT))

# Import the primitives you need. See "Available primitives" below.
from synth import (
    SR, A4, freq, t_axis, adsr, soft_clip, lowpass_fft,
    reverb_stereo, to_stereo, write_wav,
)

OUT = HERE / "samples"

# ---------- voice synthesis ----------

def voice_pluck(midi):
    """Felt-mallet pluck. Returns mono numpy array."""
    f = freq(midi); dur = 3.5
    n = int(dur * SR); t = t_axis(dur)
    sig = (
        1.00 * np.sin(2*np.pi*f*t) +
        0.30 * np.sin(2*np.pi*f*2*t) * np.exp(-t*4) +
        0.12 * np.sin(2*np.pi*f*3*t) * np.exp(-t*6)
    )
    env = adsr(n, a=0.02, d=1.0, s_level=0.0, r=2.5)
    return sig[:env.size] * env[:sig.size] * 0.6

def voice_breath(seed):
    """Filtered noise wash."""
    rng = np.random.default_rng(seed)
    dur = 2.0; n = int(dur * SR)
    noise = rng.standard_normal(n)
    bp = lowpass_fft(noise, 1500.0, order=2) - lowpass_fft(noise, 400.0, order=2)
    env = adsr(n, a=0.15, d=0.5, s_level=0.4, r=1.0)
    return bp * env * 0.3

# ---------- gen_all ----------

def gen_all():
    OUT.mkdir(exist_ok=True)
    for sub in ("pluck", "breath"):
        (OUT / sub).mkdir(exist_ok=True)

    # Pitched voice — render one sample per scale pitch, name `NN_m{midi}.wav`
    pluck_midis = [57, 59, 61, 64, 66, 69, 71, 73, 76, 78, 81]
    for i, m in enumerate(pluck_midis):
        print(f"  pluck m{m}")
        st = reverb_stereo(voice_pluck(m), wet=0.20, decay_s=2.5,
                            predelay_ms=25, brightness=0.45)
        write_wav(OUT / "pluck" / f"{i:02d}_m{m}.wav", st, target_peak=0.85)

    # Unpitched voice — name `NN.wav`. Each call uses a different seed so
    # the bag picker rotates through audibly distinct variants.
    for i in range(4):
        print(f"  breath {i}")
        mono = voice_breath(seed=i)
        st = reverb_stereo(mono, wet=0.10, decay_s=1.5,
                            predelay_ms=15, brightness=0.3)
        write_wav(OUT / "breath" / f"{i:02d}.wav", st, target_peak=0.6)

    print("done.")

if __name__ == "__main__":
    gen_all()
```

### Sample naming convention

| File              | Picker mode                       | When to use                                       |
|-------------------|-----------------------------------|---------------------------------------------------|
| `NN_m{midi}.wav`  | Markov pitched mode               | Voice has a pitch — pluck, mallet, bell, kalimba. |
| `NN.wav`          | Bag mode (random no-repeat)       | Unpitched/multi-pitch — chord pads, noise textures, bird chirps. |

The file index `NN` is just for sort order — start from `00`. The `m{midi}` suffix is what the Markov picker reads. **Pitched samples must include `_m<MIDI_int>` in the filename.**

For **tonal-overlay mode** (noise voices that walk a scale via rate-shift): name files `NN.wav` (bag mode), set `tonal_anchor_midi` in the voice config, and declare `scale_pitches` on the preset. The picker handles the rest.

## Available primitives in `synth.py`

Imports you'll commonly want from the project root `synth.py`:

| Name | What it does |
|---|---|
| `SR` | Sample rate constant (44100). |
| `A4` | A=432.0 — the project's tuning anchor. |
| `freq(midi)` | MIDI int → Hz at A=432. Use this for every pitched voice. |
| `t_axis(seconds)` | Returns `np.arange(...) / SR`. |
| `adsr(n, a, d, s_level, r, hold=0.0)` | Standard ADSR envelope with optional hold. |
| `soft_clip(x, drive=1.0)` | `np.tanh(x * drive)` — gentle warmth/limiting. Drive 1.05–1.15 is the safe range. |
| `lowpass_fft(x, cutoff_hz, order=4)` | Smooth lowpass via FFT. Use for muffled, warm timbres. |
| `make_ir(decay_s, predelay_ms, brightness)` | Synthetic stereo IR (cached). Returns (L, R) noise impulses with the supplied decay/brightness. |
| `fft_convolve(a, b)` | Mono FFT convolution. |
| `reverb_stereo(mono, wet, decay_s, predelay_ms, brightness)` | The convenient one. mono → stereo wet+dry mix. Short-circuits to dry when `wet ≤ 0.02`. |
| `to_stereo(mono, pan)` | Constant-power pan. Pan is -1.0 (left) to +1.0 (right). |
| `write_wav(path, stereo, target_peak=0.85)` | Peak-normalize and write a 16-bit stereo WAV. |

You can also write your own helpers inline if you need something more specialized (granular grains, Karplus-Strong, etc.).

## Tuning conventions

### A=432 Hz, A-rooted

The project uses A=432 Hz throughout via the `freq(midi)` helper. All shipped presets are A-rooted (their tonic is some A in some octave). Mixing presets feels musically continuous because they share that root.

You can build a non-A-rooted preset, but: (a) set `phrase_roots` to the appropriate pitch classes, (b) accept that switching from a non-A preset back to meadow/cathedral will sound like a key change.

### Scales — a starter table

| Mood | Scale | Pitch classes (from root) | Notes |
|---|---|---|---|
| Bright, satisfying | Major pentatonic | 0 2 4 7 9 | meadow's territory; can do no wrong |
| Lush modal | Lydian | 0 2 4 6 7 9 11 | cathedral's territory; raised 4 = "open" feel |
| Spacious | Lydian + 9 | 0 2 4 6 7 9 11 (use sparsely) | rainfall's territory |
| Contemplative | Hirajoshi or In Sen | 0 1 5 6 10 (In Sen) | koto's territory; minor 2nd is the gesture |
| Wistful, modal-jazz | Dorian | 0 2 3 5 7 9 10 | natural minor with raised 6 |
| Dark, cinematic | Phrygian | 0 1 3 5 7 8 10 | ♭2 is the gesture; very Spanish/Middle-Eastern |
| Exotic warm | Phrygian dominant | 0 1 4 5 7 8 10 | maj 3 over Phrygian — flamenco |
| Floaty, ambiguous | Whole-tone | 0 2 4 6 8 10 | no half-steps; no leading tone — Debussy |
| Tense, jazzy | Octatonic / Diminished | 0 2 3 5 6 8 9 11 | Messiaen's lane; alternates whole + half |
| Microtonal | Just-intonation A | 1, 9/8, 5/4, 4/3, 3/2, 5/3, 15/8 | great for sustained pad chords |

If you want a non-major-family scale, you'll likely also want `interval_weights` to favor the steps that are *actually* in the scale. The koto preset's table flips minor 2nd from 0.20 to 1.80 — that's the move.

### Reverb — per-voice taste, not blanket

A core design rule. Reverb settings live in the call to `reverb_stereo()` per voice render, not in the preset config. Per-voice policy:

| Voice character | wet | decay_s | Why |
|---|---|---|---|
| Percussive accents (wood, tap, mokugyo, shaker) | 0.0–0.02 | — | Should feel **in the room with you**. Not a cave. |
| Bird chirps, pure-noise textures | 0.05–0.10 | 1.5–2.0 | Near-dry; tiny tail. |
| Plucked melodic (pluck, mallet, kalimba, drop, koto) | 0.18–0.25 | 2.0–3.5 | Light reverb gives presence, not wash. |
| Bells, chimes, bowls | 0.35–0.45 | 4.0–5.5 | Medium tail — bell *body* needs space. |
| Pads, blooms, swells | 0.45–0.55 | 4.5–6.5 | Heavy tail — that's the character. |

`reverb_stereo()` short-circuits at `wet ≤ 0.02` and returns a true dry stereo replicate — use it freely for percussive parts.

### Bell-like voices — additive only, never FM

FM bells at high modulation indices produce buzzy, metallic sidebands. Real bells get their character from **inharmonic partials with independent decay rates**. The recipe that works (from cathedral's `voice_bell`):

```
fundamental at f                 decaying slowly  (e.g. exp(-t * 0.55))
0.50 × stretched octave at 2.005x  decaying medium (exp(-t * 0.95))
0.22 × inharmonic at 2.76x        decaying faster (exp(-t * 2.2))
0.07 × glass shimmer at 4.5x      decaying fastest (exp(-t * 4.0))
+ optional 0.18 × at 1.003x       slight detune for life
```

Avoid partials above ~5x f (they get harsh fast). Independent decay is what makes it sound real.

### Chord voicings — vary inversions

If you have a chord-based voice (pad, bloom, swell), don't render every variant in root position. The ear locks onto the lowest note as "home" — rotating through three root-position chords means every Stop event sounds identically resolved. Add at least one first-inversion (third in bass) and one second-inversion (fifth in bass) for variety. See meadow's bloom chord list for an example.

### Stretched octaves on long pads (optional)

For sustained chord voicings (≥6 s pads), equal-tempered intervals beat audibly. If you want the "expensive" pad sound, render in just intonation — the major third is 5:4 (not the equal-tempered 5/4 + 14¢), the fifth is 3:2. cathedral's drone uses just-intoned 3:2 between A2 and E3.

## What sounds bad (don't do this)

These are the failure modes the four shipping presets actively avoid. Most have an explicit memory entry from someone iterating on this project.

1. **FM bells.** Carrier:modulator ratios at 1:1.41 / 1:2.76 with index 3-6 → buzzy synth-bell sound. Use additive synthesis. Period.
2. **Reverb on everything.** Even a percussive wood tap with a 0.6 s tail makes the whole preset feel washy and distant. Be ruthless about the per-voice policy above.
3. **Click attacks.** Anything with attack time below ~5 ms will click on a sharp pluck. Use ≥5 ms minimum, ≥15 ms ideal.
4. **Harsh upper partials.** Anything above ~5x the fundamental needs to decay fast (exp(-t * 4) or steeper), or it screeches. The original koto bowl had 8.93x partials and had to go.
5. **Walking up the scale.** If your sample directory is sorted by MIDI and you don't have the Markov picker, you'll get A → B → C# → D# every time. The picker handles this — but if you're testing your render directly via `claudio voice <name> play`, it picks at random; don't conclude "the scale sounds boring" from that test.
6. **Preset gain too loud by default.** The project's calibrated default is `master_gain` 0.45–0.55. Voices typically 0.15–0.55. Err on the side of slightly-too-quiet — users can turn up.
7. **Voices fighting each other.** If your `cluster` voice is 4 seconds of dense pentatonic flourishes and your `pluck` voice fires every 0.5 s, the pluck will smear into the cluster's tail and you'll hear nothing. Use MIOI to space slow voices apart, or layer them with intent.
8. **One key too far.** A preset is one room. If you mix scales (a Phrygian voice next to a Lydian voice), pick which one is dominant — the other voices should be in *that* world or simply unpitched. Don't have two melodic voices in incompatible scales.

## How to listen-test

The implementation cycle:

```bash
# 1. Edit render.py
# 2. Render the samples
python3 presets/your-preset/render.py

# 3. Quick sanity check on one voice
claudio voice <name> play   # plays a random sample at active master gain

# 4. Walk every event mapping
claudio preset use your-preset
claudio test                # 14-event walk with 2.5s gaps

# 5. Realistic preview
claudio demo                # 60-second musical script

# 6. Iterate while listening
claudio tune                # live TUI; ←/→ adjust gain & MIOI per voice
```

The TUI (`claudio tune`) is the right place to taste-tune. It writes back to `preset.json` immediately. Once you like the levels, copy `preset.json` to `preset.default.json` (so `claudio preset reset <name>` works).

### What "good" sounds like

- A 60-second `claudio demo` should feel **like a moment**, not a checklist.
- Switch to your preset, then to one of the four shipping presets — does the transition feel like changing rooms or like an abrupt cut? Same A-root → continuous; mixed roots → discontinuous (sometimes intentional).
- Run `claudio test` with no song mode, no quant. The Markov picker should produce melodic motion, not random pitch leaps.
- After 5 minutes of normal Claude Code use, the preset should fade into the background — not because it's quiet, but because your nervous system stops tracking it as "music" and starts tracking it as "the room."

If after 5 minutes you're still consciously hearing it as a notification, gain or MIOI is too high somewhere.

## Cohesion checklist

Before you ship a preset, walk this:

- [ ] **One scale per preset.** Pitched voices share a tonal world. Unpitched voices either sit out or use tonal-overlay onto the same scale.
- [ ] **One reverb language.** All voices' reverb settings sit on the same continuum (per-voice policy table above). No wood-tap-with-cathedral-reverb.
- [ ] **Gain budget balanced.** No single voice clips when others are quiet; no single voice disappears when the mix is busy. Use the TUI to verify.
- [ ] **Slow voices have long MIOI.** Pads / blooms / clusters with `mioi: 8.0+`. Otherwise they pile up and turn to mud.
- [ ] **Bloom inversions vary.** If a chord voice has 3 voicings, at least one is an inversion. Otherwise every Stop sounds identical.
- [ ] **Failure has a different sound.** `PostToolUse.on_failure` should map to a *different timbre*, not just a different pitch. The user needs to hear that something went wrong.
- [ ] **Description matches the sound.** If `description` says "bright and happy," the preset shouldn't open with a minor 6th drone. Re-read it after listening.
- [ ] **Phrase landings are correct.** If the scale is non-A-rooted, set `phrase_roots`. Check by running `claudio test` and listening for pitches recurring at phrase boundaries.

## Sharing your preset

Once it's working:

- **For yourself:** `presets/<name>/preset.json` and `render.py` go in your project. Run `python3 install.py` to render samples. Run `claudio preset use <name>` to activate.
- **For others (PR to the public repo):** include `preset.json`, `preset.default.json` (a copy of preset.json snapshotted at the moment you decided this was good), and `render.py`. Don't commit `samples/` — the recipient regenerates with `claudio regen <name>`. Add a paragraph to the README under the "Listen first" section. Voice files go through a real listen test before merge.

---

That's it. The bar is **cohesion**, not complexity. A two-voice preset can ship if both voices belong to the same room. A ten-voice preset can fail if any one of them doesn't.

If in doubt, copy meadow's `render.py` and adjust the scale + timbres. It's the simplest of the shipping presets and the structure transfers cleanly.
